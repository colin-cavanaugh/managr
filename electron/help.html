managr

Cross-platform file management and automation for your desktop.

managr gives you a real-time view of what's inside any directory — how much space each folder occupies, which file types are present, and where they live inside the tree. You can create automation rules that move, organize, or clean up files automatically as they appear, and roll back any operation with one click.

Quick start: Open the Explorer tab, navigate to any folder, and the File Types breakdown populates immediately. Hover an extension like .ba2 to see what it is, or click it to filter the file list to only that type.

File Types Panel

The left panel in the Explorer shows every extension found in the current directory, ranked by total size. Each row includes:

  • Size bar — visual relative footprint compared to the largest type
  • Count · Size — number of files and combined size
  • + Rule — opens a quick-rule dialog pre-filled for that extension

Extension Tooltips

Hover over any extension to see a plain-English description. managr ships with descriptions for 300+ extensions, from common formats to niche ones:

  • .ba2 — Bethesda Archive 2 — game data for Fallout 4, Starfield
  • .gguf — Quantized LLM model format used by llama.cpp / Ollama
  • .parquet — Columnar data format for big data / analytics

If an extension has a known description, its label shows a help cursor. Unknown extensions show no tooltip.

Volume Loading (Calculate Volume)

Folder sizes are loaded one at a time in the background so the UI stays responsive. The status appears in the sort bar:

  • Pause Volume — stops loading if you don't need sizes yet
  • Calculate Volume — appears at drive roots and after pausing; click to start or resume
  • ✓ Volume — all sizes in the current view are loaded
Session cache: Folder sizes are remembered for your entire session. Navigating away and back to a directory loads instantly from cache — no re-scanning.

At drive roots (C:\, /mnt/c, /) auto-scanning is disabled by default — use Calculate Volume to start manually.

Deep Scan vs Calculate Volume

These are two different operations with different scope:

Feature Calculate Volume Scan All Subfolders
Folder sizes Top-level subfolders only Every nested folder, cached globally
Extension breakdown Files in current folder only Entire subtree recursively
Extension filtering for folders Not available Available — folders shown if they contain that type
Speed Fast (parallel stat calls) Slower on large directories; can take minutes
Stop mid-scan Pause / Resume Stop Deep Scan (top-right of sort bar)
After a deep scan, all discovered folder sizes are added to the session cache. Navigating into any subfolder that was scanned will have its children's sizes available immediately.

Extension Filter

Click any extension in the File Types panel to filter the Contents list to only show items of that type. An active filter displays as a pill in the Contents header:

.ba2 — Bethesda Archive 2 ×

  • Click the pill (or the extension row again) to clear the filter
  • Hover the pill to see the full description

Filtering with and without Deep Scan

Without Deep Scan

Only files with that extension are shown. Folders are hidden because their contents aren't known. A hint bar offers a one-click shortcut to run Deep Scan and enable folder filtering.

With Deep Scan

Folders are also shown if they contain at least one file of that extension anywhere inside them — no matter how deeply nested. This lets you instantly see which directories are home to that file type.

Search

  • Filter by name — instantly filters the visible list by filename (no server call, updates as you type)
  • Deep search — sends a recursive server-side search across all subdirectories, with 400ms debounce to avoid spamming

Deep search and extension filter can be used together: filter by .png first, then type a name pattern to narrow within that set.

Automation Rules

Rules watch a directory and automatically act on files when they match a set of conditions.

Parts of a rule

  • Trigger — when the rule fires: file created, file modified, or manual
  • Conditions — what must match: extension, name pattern, size threshold
  • Actions — what to do: move, copy, rename, delete, or back up

Quick rule from Explorer

Click + Rule on any extension row to open a dialog pre-filled with that extension as the condition and the current directory as the watch path. Fill in a destination and click Create Rule — done.

Running rules

Enable the file watcher from the Watcher section to have rules fire automatically in real time. Or run any rule manually from the Rules page with the Run button.

Snapshots & Rollback

Every file operation — whether from a rule or manual action — is logged in the Activity page. Bulk operations (rule runs, bulk move, bulk delete) create a snapshot batch.

From the Snapshots page, click Rollback on any batch to restore all files to their original locations in one click. Individual activity entries on the Activity page can also be inspected to see exactly what moved where.

Note: Rollback moves files back — it doesn't undo renaming of the target directory itself. If a destination folder was also renamed or deleted after the operation, rollback will attempt the move but may report an error.

Pinned & Frequent Directories

  • Pinned — manually bookmark any directory from the sidebar. Pinned directories persist across sessions.
  • Frequent — the four most-visited directories are automatically surfaced in the sidebar, based on how many times you've navigated to them.

Skip Folders

Mark a folder as skipped using the ⊘ button in its action row. Skipped folders are:

  • Excluded from volume loading (size shown as "skipped")
  • Excluded from deep scans and extension analysis
  • Excluded from watcher events and rule triggers

Useful for virtual filesystems, package caches (node_modules), or system folders you don't want included. Click ⊘ again on a skipped folder to unskip it.
